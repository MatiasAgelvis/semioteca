# Semioteca Backend

The backend processes source documents (ODT/DOCX files) into a structured card collection with automatic semantic tagging.

## Architecture

The backend consists of three main stages:

1. **Card Generation** (`generate_cards_json.py`) - Extract and structure content from source documents
2. **Tagging** (`tag_cards.py`) - Apply semantic tags to cards based on content
3. **Data Models** - Unified representation of cards, books, and the library

## Scripts

### `generate_cards_json.py`

Converts ODT/DOCX source documents into a structured JSON card collection.

**Usage:**
```bash
python generate_cards_json.py [OPTIONS]
```

**Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `--source-dir` | string | `ODT` | Directory containing source `.odt` or `.docx` files |
| `--output-json` | string | `cards.json` | Path for the generated JSON collection |
| `--image-dir` | string | `cards_images` | Directory to store extracted images from documents |
| `--verbose` | flag | - | Print progress information during generation |
| `--report-anomalies` | boolean | `true` | Print suspiciously short or long card warnings |
| `--anomaly-sigma` | float | `4.0` | Z-score threshold for outlier detection (relative to source mean) |
| `--anomaly-min-chars` | int | `25` | Always flag cards at or below this character count |
| `--anomaly-max-chars` | int | `100000` | Always flag cards at or above this character count |
| `--anomaly-source-share` | float | `0.12` | Flag cards consuming at least this share (12%) of source characters |
| `--anomaly-relative-ratio` | float | `4.0` | Require outliers to be N times larger/smaller than source mean |

**How it works:**

1. Loads source document configurations from `source_documents.py`
2. Converts ODT files to DOCX (via pypandoc) then extracts text and images
3. Splits documents into cards using regex patterns defined in each source config
4. Validates card metadata and reports length anomalies
5. Outputs `cards.json` organized by book (title, author, year)

**Example:**
```bash
python generate_cards_json.py --source-dir ODT --verbose --output-json cards.json
```

### `tag_cards.py`

Applies semantic tags to all cards in a dataset. Supports multiple tagging strategies.

**Usage:**
```bash
python tag_cards.py
```

**Configuration:**
- Uses `cards.json` as input (generated by `generate_cards_json.py`)
- Tagger strategy can be switched between `nli_tagger` (default) and `embedding_tagger`
- Outputs updated `cards.json` with tags assigned to each card

**Tagging Strategies:**

- **NLI Tagger** (`nli_tagger.py`) - Zero-shot classification using entailment models
  - Model: Cross-encoder based NLI
  - Supports custom confidence thresholds
  - Default: max 2 tags per card, 0.5 confidence threshold

- **Embedding Tagger** (`embedding_tagger.py`) - Semantic similarity via embeddings
  - Model: `ibm-granite/granite-embedding-311m-multilingual-r2`
  - Default: max 2 tags per card, 0.8 similarity threshold

### `nli_tagger.py` & `embedding_tagger.py`

Alternative tagging implementations. See `tag_cards.py` to switch between them.

### `tags.py`

Manages tag definitions loaded from `card-tags.json`.

**Key utilities:**
- `load_tag_definitions()` - Load available tags
- `CardTag` dataclass - Tag name + description
- `labels_to_name_dict()` - Convert display labels to tag names

### `source_documents.py`

Defines source document configurations in the `SourceDocument` enum.

Each entry specifies:
- `filename` - ODT/DOCX file to process
- `split_pattern` - Regex pattern to split document into cards
- `title`, `author`, `book`, `year` - Curated metadata (hand-verified)

**Available patterns:**
- `PAGE_DOT_PATTERN` - For documents with markers like "p. 45"
- `PARENTHESIS_YEAR_PAGE_PATTERN` - For citations like "AUTHOR (1990:45)"
- `LEVINSON_2004_PATTERN` - Special pattern for mixed citation styles
- `WARNOCK_PAGE_PATTERN` - For running headers with page numbers

All cards inherit metadata from their source config—no per-card overrides to prevent data drift.

## Data Models (`card_models.py`)

```python
Card          # Individual content snippet with metadata (title, author, year, page, tags)
Book          # Collection of related cards (groups by title/author/year)
Library       # Top-level container (list of books)
ImageRef      # Embedded image metadata and paths
CardSection   # Intermediate parsing result (marker, page, year, content)
```

## Setup

### Requirements

```bash
pip install -r requirements.txt
```

Key dependencies:
- `mammoth` - DOCX text extraction
- `pypandoc` - ODT → DOCX conversion
- `sentence-transformers` - Embedding models (for embedding tagger)
- `torch` - Deep learning (for NLI/embedding taggers)
- `python-dotenv` - Environment variables

### Configuration

1. Add source ODT/DOCX files to the `ODT/` directory
2. Register each file in the `SourceDocument` enum in `source_documents.py` with:
   - Filename
   - Split regex pattern
   - Curated metadata (title, author, book, year)
3. Define tags in `card-tags.json`:
   ```json
   [
     {"name": "tag-slug", "description": "Human-readable description"},
     ...
   ]
   ```
4. Create `.env` file if using embedding/NLI taggers (requires `HF_TOKEN` for Hugging Face)

## Workflow

```
ODT/DOCX sources
     ↓
[generate_cards_json.py]  →  cards.json (untagged)
     ↓
[tag_cards.py]  →  cards.json (tagged)
     ↓
(Frontend reads from backend/cards.json)
```

## Anomaly Detection

When generating cards, the script identifies potential issues:

- **Tiny cards** - Below 25 characters (configurable)
- **Huge cards** - Above 100,000 characters (configurable)
- **Z-score outliers** - Cards 4+ standard deviations from source mean
- **Dominant cards** - Cards consuming 12%+ of source characters

Use `--report-anomalies` to inspect these and refine split patterns or source configs.

## Notes

- **Metadata is curated** - All cards inherit `title`, `author`, `year` from the source config, never from parsed content. This prevents contamination from cross-references.
- **Images are extracted** - DOCX/ODT images are saved to `cards_images/` with relative paths in card metadata.
- **Regex patterns are source-specific** - Each source has its own split pattern to handle different citation styles.
